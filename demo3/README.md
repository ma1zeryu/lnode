## 基础知识

```
请求1 → 事件循环 → 处理请求 → 返回响应
请求2 → 事件循环 → 处理请求 → 返回响应  （复用同一线程）
请求3 → 事件循环 → 处理请求 → 返回响应
```

优势：

+ 单线程事件循环，避免频繁切换的开销
+ 非阻塞I/O，提高吞吐量
+ 跨平台
+ 生态好

| 特性         | 浏览器 JavaScript     | Node.js JavaScript    |
| :----------- | :-------------------- | :-------------------- |
| 全局对象     | `window`              | `global`              |
| 模块系统     | ES6 modules, AMD      | CommonJS, ES6 modules |
| 文件系统访问 | 不可访问              | 完全访问              |
| 网络请求     | XMLHttpRequest, Fetch | http, https 模块      |
| DOM 操作     | 支持                  | 不支持                |
| 进程控制     | 不支持                | 支持                  |

自身内置了HTTP服务器，可以直接创建服务器以及接受和响应请求
